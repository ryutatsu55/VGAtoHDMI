# VGAtoHDMI — ピクセルデータ仕様

カメラ等の外部ソースからBRAMにピクセルデータを書き込み、HDMIモニタに出力するためのリファレンスです。

---

## システム構成

```
外部データソース (カメラ等)
        │  Port A (clock50, 書き込み)
        ▼
  ┌─────────────┐
  │ BRAM 2-Port │  8bit × 307200 words
  └─────────────┘
        │  Port B (clock25, 読み出し)
        ▼
    PVI.v (ピクセルカウンタ)
        │
        ▼
  ADV7513 → HDMI出力 (640×480 @ 60Hz)
```

---

## BRAMの仕様

| 項目 | 値 |
|------|-----|
| データ幅 | 8ビット（グレースケール: 0x00=黒, 0xFF=白） |
| 深さ | 307,200 words |
| アドレス幅 | 19ビット `[18:0]` |
| Port A クロック | clock50（50 MHz） |
| Port B クロック | clock25（25.175 MHz） |
| Port B 出力 | UNREGISTERED（組み合わせ出力） |

---

## アドレスマッピング

画面座標 `(H, V)` に対するBRAMアドレスは以下の式で計算します。

```
addr = V × 640 + H
```

| 座標 | アドレス |
|------|---------|
| (0, 0)     — 左上 | 0x00000 (0) |
| (639, 0)   — 右上 | 0x0027F (639) |
| (0, 1)     — 左端2行目 | 0x00280 (640) |
| (0, 239)   — 垂直中央左 | 0x1DF80 (122,880) |
| (319, 239) — 画面中央 | 0x1E13F (123,199) |
| (639, 479) — 右下 | 0x4AFFF (307,199) |

```
H : 0 ～ 639（左→右）
V : 0 ～ 479（上→下）
addr : 0 ～ 307,199
```

ハードウェアでの計算（乗算なし）:

```verilog
addr = (V << 9) + (V << 7) + H
//   = V×512  + V×128  + H
//   = V×640  + H
```

## Port A（書き込み）インタフェース

`bram_2port` のPort A信号：

| 信号名 | 方向 | 幅 | 説明 |
|--------|------|-----|------|
| `clock_a` | input | 1 | 50 MHz クロック（clock50） |
| `address_a` | input | [18:0] | 書き込みアドレス |
| `data_a` | input | [7:0] | 書き込みデータ（グレースケール） |
| `wren_a` | input | 1 | 書き込みイネーブル（High で書き込み） |
| `q_a` | output | [7:0] | （未使用） |

### 書き込みタイミング

- `wren_a` が High の立ち上がりエッジで `address_a` のアドレスに `data_a` が書き込まれます。
- 連続アドレスに書き込む場合はアドレスを毎サイクルインクリメントしてください。

### Verilogでの記述例

```verilog
// 1フレーム分を順番に書き込む例
reg [18:0] wr_addr;
reg [7:0]  wr_data;
reg        wr_en;

// アドレス計算
assign wr_addr = ({10'b0, v_cnt} << 9) + ({10'b0, v_cnt} << 7) + h_cnt;

// BRAMへ接続
bram_2port bram_2port (
  .clock_a   (clock50),
  .address_a (wr_addr),
  .data_a    (wr_data),
  .wren_a    (wr_en),
  .q_a       (),
  // Port B は PVI.v 側が使用
  .clock_b   (clock25),
  .address_b (bram_addr_b),
  .data_b    (8'h00),
  .wren_b    (1'b0),
  .q_b       (bram_rdata_b)
);
```

## カメラデータ取得の確認方法

カメラからGPIO経由でグレースケールデータを取得できているか確認する手順です。

### 1. pixel_gen.v を差し替える

`pixel_gen.v` の ST_WRITE ステートでカメラから取得したデータを書き込むように変更します。

```verilog
// pixel_gen.v の bram_wdata をカメラデータに置き換える例
bram_wdata <= camera_pixel_data;  // GPIOから受け取った8bitグレースケール値
```

### 2. 確認パターンの書き込み

まず `pixel_gen.v`（バウンシングボール）をそのまま動かし、モニタに映像が出ることを確認します。
次にカメラデータに切り替えて以下を確認します：

- **画面全体が黒** → カメラデータが常に `0x00`（データ線がLow固定の可能性）
- **画面全体が白** → カメラデータが常に `0xFF`（データ線がHigh固定の可能性）
- **水平方向に縞模様** → 水平同期（H方向のアドレス）が正しく動いている
- **垂直方向に縞模様** → 垂直同期（V方向のアドレス）が正しく動いている
- **画像が横にずれる** → `H_start` のオフセットがずれている
- **画像が縦にずれる** → `V_start` のオフセットがずれている

### 3. アドレスの対応確認

フレームの先頭（左上ピクセル）が `address_a = 0` になっていることを確認してください。
カメラの水平ピクセルカウンタ・垂直ラインカウンタとアドレスのタイミングがずれている場合、
画像が回転・反転して見えることがあります。

## 注意事項

- Port A と Port B が同じアドレスに同時アクセスすると `BIDIR_DUAL_PORT` モードでは
  `NEW_DATA_NO_NBE_READ` の動作になります（書き込みデータが読み出されます）。
  書き込み中に画面のその部分が読まれても壊れることはありませんが、一瞬ちらつく場合があります。
- Port B の出力は **UNREGISTERED**（組み合わせ出力）です。
  `clock_b` の立ち上がりでアドレスが確定し、直後に出力が変化します。
- データはグレースケール8ビットです。カラーカメラの場合は輝度 `Y = 0.299R + 0.587G + 0.114B`
  などで変換してから書き込んでください。
